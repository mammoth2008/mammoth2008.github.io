### 2.2 分布式计算与云计算
### 云计算 - 概念更抽象 - 用户需求封装成服务 - 提供统一窗口 - 解决人与 IT 资源的关系问题
### 概念更抽象 - 将复杂的计算任务抽象成服务 - 将底层计算资源抽象成服务 - 用户只需关心输入输出 - 无需关心服务细节
![Cloud Computing](img/c02/cloud-computing.webp) #### Cloud Computing
### 用户需求封装成服务 - 存储服务 - 计算服务 - 数据库服务 - 人工智能服务 - 用户根据需求选择服务 - 灵活调整服务规模和性能
### 提供统一服务窗口 - 在一个平台上管理和使用所有的云服务 - 简化用户操作
![Aliyun](img/c02/aliyun.png) #### Aliyun
![Aliyun](img/c02/aliyun1.png) #### Aliyun
### 解决人与 IT 资源的关系问题 - 用户无需了解硬件和软件 - 用户无需配置和管理 - 简化人与 IT 资源的关系
### 2.3 云计算关键技术
### 分布式海量数据存储 - 大数据技术 - 对象存储技术 - 块存储技术
### 传统存储的问题 - 数据库容量有限 - 并行取数困难 - 数据库接口访问效率低 - 并发访问多 - 扩容成本高
### 数据存储技术 - 分布式文件系统 - Google File System - Hadoop Distributed File System
![Google File System](img/c02/gfs.png) #### Google File System
### 分布式对象存储 - 云盘 - Amazon S3 - OpenStack Swift
### 分布式块存储 - Ceph - GlusterFS
### 虚拟化技术 - 硬件, 软件, 数据隔离 - 资源, 网络,应用虚拟化 - IT 架构动态化 - 资源集中管理 - 资源动态分配 - 提升适应需求和环境的能力
### 虚拟化技术的特点 - 资源分享 - 资源定制 - 细粒度资源管理
### 云管理平台技术 - 大量服务器协同 - 方便部署 - 故障快速发现和恢复 - 自动化 - 智能化
### 并行编程技术 - 数据并行 - 消息传递
### 数据并行模型 - 层次较高 - 数据分布在不同的节点 - 每个节点处理自己的数据 - 节点之间通信 - 提供全局地址空间 - 用户列出并行操作和对象
![Data Parallelism](img/c02/data-parallelism.png) #### Data Parallelism
### 消息传递模型 - 面向分布式内存或共享内存 - 控制更灵活 - 通信更细粒度
### 数据管理技术 - 分布式数据库 - BigTable - HBase
### 2.4 云计算的优势与挑战
### 云计算的优势 - 超大规模 - 虚拟化 - 高可靠性 - 通用性 - 高可扩展性 - 按需服务 - 便利性
### 云计算的挑战 - 服务的持续可用性 - 服务的安全性 - 服务的迁移 - 服务的性能
