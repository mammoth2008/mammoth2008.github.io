### 3.1 云基础设施机制
![Cloud Infrastructure](img/c03/cloud-infrastructure.png) #### Cloud Infrastructure
### 云基础设施机制 - 云环境的基础构件块 - 面向计算, 存储, 网络 - 虚拟网络边界 - 虚拟服务器 - 云存储设备 - 就绪环境
### 虚拟网络边界 - 网络设备 - 作为虚拟化环境部署 - 隔离授权用户 - 隔离 IT 资源 - 隔离可用带宽
### 虚拟防火墙 - 在单一硬件平台上提供多个防火墙实体 - 独立维护一组安全区域 - 独立维护一组资源对象 - 独立维护包安全策略 - 独立维护安全规则 - 可限制所用资源数
### 虚拟专用网络 (VPN) - 在公共网络上建立私有网络 - 通过加密和隧道技术实现 - 提供安全保障 - 提供服务质量保证 (QoS) - 可扩充性和灵活性 - 提高可管理性
### 虚拟服务器 - 虚拟机监视器 (VMM) - 虚拟化平台 (Hypervisor) - 虚拟机 (VM) - 多个用户同享一个物理服务器 - 多实例 - 隔离性 - 封装性
### 虚拟服务器的优点 - 实时迁移 - 快速部署 - 高兼容性 - 提高资源利用率 - 动态调度资源
### 云存储设备 - 为云设计的存储设备 - 文件存储 - 块存储 - 数据集存储 - 对象存储
### 用户操作安全 - 用户身份验证 - 文件覆盖问题 - 版本管理 - 数据备份
### 服务端安全 - 数据加密 - 数据完整性 - 数据可用性 - 数据备份
### 就绪环境 - 云环境的基础构件块 - 已安装好的 IT 资源 - 数据库 - 中间件 - 应用程序 - 开发工具
### 3.2 云管理机制
### 云管理 - 私有云或混合云 - 硬件差异 - 软件差异 - 环境差异 - 需求差异
### 云管理 - 资源池规划 - 服务目录管理 - 云配置管理数据库 - 流程管控 - 资源监控 - 资源调度
### 传统管理 - 管理对象: 网络，存储，服务器，OS，应用等 - 管理目标: 实现 IT 系统的正常运作 - 管理特色: 专业技能，手动管理，竖井式管理 - 管理平台的易用性: 低 - 管理规模: 数百量级 - 用户: 管理员 - 整合: 基于事件，数据库，私有接口 - 管理手段: 离散工具
### 云管理 - 管理对象: IaaS，PaaS，SaaS 等云服务 - 管理目标: 实现云服务端对端交付及数据中心运维 - 管理特色: 屏蔽底层细节，自动化，多租户，共享管理平台 - 管理平台的易用性: 高 - 管理规模: 数万量级 - 用户: 分层管理，多租户 - 整合: 面向服务 - 管理手段: 充分自动化
### 远程管理系统 - Remote Management System - 使用与管理入口: 管理不同的基于云的 IT 资源 - 自助服务入口: 用户自主选择服务并提交请求
### 远程管理系统的功能 - 为按需云服务提供和释放 IT 资源 - 监控云服务的状态，使用和性能 - 监控 QoS 和 SLA 的实行 - 管理租赁成本和使用费用 - 管理用户账户，安全凭证，授权和访问控制 - 追踪对租赁服务内部与外部的访问 - 规划和评估 IT 资源供给 - 容量规划
### 资源池化管理 - Resource Pooling Management System - 考虑资源种类: 计算，存储，网络，应用等 - 考虑应用架构: Web 层，应用层，数据层等 - 应用等级保障: 金，银，铜等 - 管理需求: 生产区，同城灾备区，异地灾备区等 - 安全域: 安全标准，隔离标准等
### 服务等级协议管理系统 - Service Level Agreement，SLA - 保证可计量的性能指标 - SLA Monitor 用来观测性能指标
### 计费管理系统 - Billing Management System - 涉及云提供者的结算 - 涉及云用户的计费 - 付费方式有预付费和后付费 - 计费方式有按需计费和包年包月计费
### 资源备份 - Resource Backup - 对同一个 IT 资源建立多个实例 - 加强 IT 资源的可用性和可靠性 - 公有云平台往往提供异地多副本备份
### 云监控 - Cloud Monitor - 保证应用和服务的性能 - 估算工作负荷 - 追踪资源使用情况 - 帮助规划资源供给
### 云监控系统的功能 - 数据采集: 容量，性能，使用情况等 - 数据处理和分析: 数据聚合，数据分析，数据挖掘等 - 告警事件管理: 告警规则，告警事件，告警通知等
### 自动化运维 - Automation Operation and Maintenance - 基于策略的自动化运维方案 - 配置: 自动化配置，自动化部署等 - 合规: 定义和应用配置策略以满足合规性要求 - 补丁: 自动化补丁管理，自动化补丁部署等 - 自动发现: 自动发现 IT 资源，自动发现配置项等
### 服务模板管理 - Service Template Management - 提供服务的部署视图 - 说明服务所需的资源 - 说明服务的配置要求 - 支持服务前端的高度灵活
### 云配置管理数据库 - Cloud Configuration Management Database，CCMDB - 存储配置信息 - 动态基础架构的数据管理 - 自动发现与自动收集
### 服务目录管理 - Service Catalog Management - 支持对服务的生命周期管理 - 定义了 IT 服务的使用者与资源之间的接口 - 管理服务实例 - 审批设定
### 租户与用户管理 - Tenant and User Management - 逻辑区分 - 物理隔离 - 权限设计 - 配额管理
### 容量规划及管理 - Capacity Planning and Management - 容量分析与规划 - 抽取性能数据 - 关联度分析预测 - What-if 分析
### 3.3 特殊云机制
### 特殊云机制 - 自动伸缩监听器 - 负载均衡器 - 故障转移系统 - 虚拟机监控器 - 资源集群 - 多设备代理 - 状态管理数据库
### 自动伸缩监听器 - Auto Scaling Listener - 监听和追踪 IT 资源使用情况
### 负载均衡器 - 非对称分配 - 负载优先级 - 上下文感知的分配
### 负载均衡器 - 软件负载均衡技术 - 硬件负载均衡技术 - 本地负载均衡技术 - 全局负载均衡技术 - 链路集合负载均衡技术
### 故障转移系统 - 主动 - 主动 - 主动 - 被动
### 资源集群 - 服务器集群 - 数据库集群 - 大数据集集群
### 多设备代理 - 作为网关存在 - 用户可以创建
### 状态管理数据库 - 存储软件的状态数据 - 降低程序占用的运行时内存量
### 3.4 虚拟化
### 虚拟化 - 计算机在虚拟的基础上而非真实的物理硬件上运行 - 优化资源 - 简化软件的重新配置
### 虚拟化架构 - 在一个物理硬件机器上同时运行多个不同的独立虚拟系统 - 提供虚拟系统资源 - 进行主机和虚拟机之间的调度 - 提供虚拟机之间的通信
### 虚拟机的用途 - 研发与测试 - 服务器合并 - 高级虚拟主机
### 虚拟化的优势 - 提高资源利用率 - 降低成本节能减排 - 统一管理 - 提高安全性
### 3.5虚拟化的分类
### 服务器虚拟化 - 完全虚拟化 - 准虚拟化 - 操作系统虚拟化
### 网络虚拟化 - 虚拟局域网 - 虚拟专用网
### 软件定义网络 - Software Defined Network - 分离控制和转发 - 控制集中化 - 网络可执行程序化行为
### 网络功能虚拟化 - Network Function Virtualization - 多种网络设备类型融合 - 使用通用设备取代专用设备
### 存储虚拟化 - 将存储设备结合起来使用 - 存储池 - 将物理存储实体和存储的逻辑表示分开
### 应用虚拟化 - 应用软件虚拟化 - 桌面虚拟化
### 应用软件虚拟化 - 将应用软件从操作系统中分离 - 通过可执行文件夹运行 - 无需设备驱动程序
### 桌面虚拟化 - 将人机交互逻辑和计算逻辑隔离 - 客户端通过网络连接到应用服务器 - 计算逻辑从本地迁移到后台
