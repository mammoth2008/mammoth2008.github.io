1.7 项目管理使用的主要工具 ### 甘特图(Gantt Chart) - ### 也称横道图(Bar Chart)，用于计划和跟踪项目活动 ### 是图和表相结合的时间线性图，通常用于显示简易资源进度计划 ### 横坐标表示时间，项目活动在图的左侧纵向排列，以活动所对应的横道位置表示活动的起始时间，横道的长短表示持续时间的长短 1.7 项目管理使用的主要工具 ### 甘特图(Gantt Chart) - ### 甘特图的优点 - 清楚地表达活动的开始时间、结束时间和持续时间，易于理解 - 使用方便，制作简单，不需要为此做人员培训 ### 甘特图的缺点 - 很难表达工程活动之间复杂的逻辑关系 - 不能表示活动的重要性 ### 甘特图的应用范围 - 可以直接用它排工期计划 - 可以用来做总体计划 - 可能会更清晰地描述项目的特定方面 1.7 项目管理使用的主要工具 ### 单代号网络图 (Activity-On-Node, AON) - ### 又称紧前关系绘图法(Precedence Diagramming Method, PDM) ### 使用方格或圆形节点表示活动，并用表示依赖关系的箭线连接节点构成 ### 依赖关系 - 完成对开始完成对完成开始对开始开始对完成 ### 绘制要求 - 不能有相同编号的节点 - 不能出现违反逻辑的表示 - 不允许有多个首节点和多个尾节点 1.7 项目管理使用的主要工具 1.7 项目管理使用的主要工具 1.7 项目管理使用的主要工具 ### 双代号网络图 (Activity-On-Arrow, AOA) - ### 又称箭线绘图法(Arrow Diagramming Method, ADM)，是一种利用箭线表示活动的项目进度网络图绘制法 ### 只使用完成对开始的依赖关系 - ### 双代号网络图的绘制要求： - 只允许有一个首节点和一个尾节点 - 一般情况下不允许出现环路 - 没有相同编号的节点，两根箭线不能有相同首节点或尾节点 - 不能出现错画、漏画 - 1.7 项目管理使用的主要工具 ### 双代号网络图 (Activity-On-Arrow, AOA) - ### 甘特图与网络图的主要差异： - 甘特图描述任务的工期，网络图描述任务之间的时间依赖 - 甘特图描述任务的时间重叠情况，而网络图既不显示时间重叠，也不显示任务的并行完成情况 - 某些类型的甘特图可以描述最早开始及最迟结束时间之间的浮动时间，网络图则借助活动矩形框内的数据来描述这一点 ### 网络图的应用范围： - 定义清晰，有明确的起始点和终结点 - 可独立于其它任务来完成 - 有序 - 1.7 项目管理使用的主要工具 ### 关键路径分析法 (Critical Path Method, CPM) - ### 是利用进度模型时使用的一种进度网络分析技术 ### 可以显示不同任务之间复杂的依赖关系，还可以估算任务的持续时间，找出按时完成任务的关键活动 ### 不能用在资源进度计划上 ### 最早开始时间(Early Start Date, ES) - 根据进度网络逻辑、数据日期以及任何进度方面的制约因素，某计划活动可能开始的最早时间点 ### 最早完成时间(Early Finish Date, EF) - 根据进度网络逻辑、数据日期以及任何进度方面的制约因素，某计划活动可能完成的最早时间点 1.7 项目管理使用的主要工具 1.7 项目管理使用的主要工具 ### 关键路径分析法：最早开始和最早完成时间计算 - ### 对于一开始就进行的活动，其最早开始时间为0 ### 某项活动的最早开始时间，必须等于或晚于直接指向这项活动的所有活动的最早完成时间中的最大值 ### 计算每项活动的最早开始时间和最早完成时间，以项目预计开始时间为参照点进行正向推算 ### 对于中间的活动，其活动的最早开始时间就是其前置活动的最早完成时间中的最大值 ### 根据项目的最早开始时间来确定项目的最早完成时间，最早完成时间可在这项活动最早开始时间的基础上，加上这项活动的工期DU(Duration)估计进行计算 ### EF = ES + DU 1.7 项目管理使用的主要工具 1.7 项目管理使用的主要工具 ### 关键路径分析法：最迟开始和最迟完成时间计算 - ### 最可能持续时间 - 所有条件都已给定时，该计划活动的持续时间 ### 乐观估计时间 - 最有利的组合时估算出来的持续时间 ### 悲观持续时间 - 最不利的组合时估算出来的持续时间 ### 设乐观估计Ta、悲观时间Tb、正常时间Tm，假设三个时间服从β分布，得出各项活动的时间平均值 ### T = (Ta+4Tm+Tb)/6 1.7 项目管理使用的主要工具 ### 计划评审技术(Program Evaluation and Review Technique, PERT) - ### 在执行项目进度安排时，最困难的活动是确定WBS中每项任务的工期计划评审技术就是为了解决这个问题 ### 术语“PERT图”有时也用来指网络图，“PERT估算”就是在确定三种估算的基础上做出的 ### PERT还可以应用于费用管理、采购管理等多个方面 ### PERT和关键路径法的本质区别在于： - 关键路径法假设项目完成的时间是确定的，它侧重于活动 - PERT估计项目在某个时间内完成的可能性，它侧重于事件 1.7 项目管理使用的主要工具 ### 进度压缩 - ### 在不改变项目范围、满足进度制约条件、强加日期或其他进度目标的前提下，缩短项目的进度时间 - ### 进度压缩的技术有： - 赶工：对费用进行进度权衡，确定如何在尽量少增加费用的前提下，最大限度的缩短项目所需的时间 - 快速跟进：在使用这种进度压缩技术时，通常同时进行着按先后顺序的阶段或活动 1.7 项目管理使用的主要工具 1.7 项目管理使用的主要工具 ### 挣值分析法(Earned Value Method, EVM) - ### 通过计划工作预算成本(Budgeted Cost for Work Scheduled, BCWS，或计划值Plan Value, PV)、已经完成工作实际成本(Actual Cost for Work Performed, ACWP或Actual Costs, AC)和实际完成活动的预算价值即挣值(Earned Value, EV，或已完成工作量的预算成本Budgeted Cost for Work Performed, BCWP)的比较，可以确定成本进度是否按计划执行 ### PV = BCWS = 计划工作量✕计划单价 ### AC = ACWP = 已完成工作量✕实际单价 ### EV = BCWP = 已完成工作量✕计划单价 1.7 项目管理使用的主要工具 ### 挣值分析法(Earned Value Method, EVM) - ### 费用偏差(Cost Variance, CV) = EV - AC - CV为0，表示实际成本消耗等于预算值大于0，表示实际消耗低小于0，表明成本超支 ### 进度偏差(Schedule Variance, SV) = EV – PV - SV为0，表示实际进度与计划进度一致大于0，表示进度提前，小于0，表示进度延后 ### 成本绩效指数(Cost Performance Index, CPI) = EV / AC - 大于1，表示实际消耗低小于1，表明超支 ### 进度绩效指数(Schedule Performance Index, SPI) = EV / PV - 大于1，表示进度提前小于1，表示进度延后 1.7 项目管理使用的主要工具 1.7 项目管理使用的主要工具 ### 挣值分析法(Earned Value Method, EVM) - ### 在分析时点上，CV小于0，SV小于0，表示进度和成本计划执行不佳，成本超支 ### 在实际成本和进度的前提下，重估项目总预算： - 未完工部分如果按目前实际的效率开展，则完成整个项目所需要的期望成本(Estimate at Completion, EAC) = AC + ( 总预算 - EV ) / CPI - 未完工部分如果按照原计划的效率进行，则EAC = AC + ( 总预算 - EV ) - 还可以根据现有条件的约束，考虑风险因素，重新估计项目未完工部分的成本，加上实际成本AC，得到EAC 1.7 项目管理使用的主要工具 ### 项目管理软件 -
